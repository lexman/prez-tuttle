Astuces d'utilisateur

Pour aller plus loin que la doc...

  • Le .tuttle
  • Les logs
  • Le code

Le .tuttle

  • Pas de serveur !
  • Le rapport montre les erreurs
  • Les resources présentes indiquent l'état de l'exécution
  • Le .tuttle se copie (par exemple depuis un serveur de build)

Les logs

  • Accessible depuis le rapport
  • Pour chaque donnée produite, les logs en rapport sont accessibles indéfiniement

Le code

  • Tout code exécuté est disponible sous forme de fichier
  • Les numéros de ligne d'erreur, c'est pas encore ça...
  • Avec tuttle c'est sans engagement !

Etendre tuttle

Ajouter une resource

  • exemple
  • Implementer la méthode exists()
  • Implementer la méthode signature()
  • Implementer la méthode remove()

Ajouter un processeur

  • exemple
  • Implementer la méthode run(self, process, reserved_path, log_stdout, log_stderr)
  • Implementer la méthode static_check(self, process)

Un système de configuration

  • Définir des constantes
  • S'appuyer sur les variables d'environement
  • Certains process ou ressources dépendent de constantes
  • Utiliser les constantes dans les process

Découvrir des dépendences à l'exécution

  • Pour utiliser certaines constantes
  • Pour les imports de sources en langage dynamique (python)

Tout plein d'affinages

  • Un processeur "auto" qui comprenne les moulinages de marmotte
